record-my-deploy

Posted on 2020-10-24

哈哈，好像成功了，
记录一下

1 创建一个repo，repo name我的就是facefall.github.io
2 接着设置为public（private好像不行）
3 在本地环境，安装好npm
4 通过npm包管理器，安装hexo
`npm i -g hexo`
5 通过hexo 生成一个博客

参考hexo官方文档

建立好之后，主要会使用的命令有一下四条

6 如果hexo server 在本地环境下能顺利运行，访问localhost:4000应该能看到效果

这里有一个坑。关于next主题的配置（也就是我目前使用的theme）

通过hexo生成的站点，使用的主题是landscape，在themes文件夹内可以看到，如果想要使用next主题，
需要去https://github.com/theme-next/hexo-theme-next

这个git repo，将主题文件clone下来或者download下来，放在themes文件夹下
更多信息去查看next主题的文档
http://theme-next.iissnan.com/getting-started.html

（themes下面一个主题一个单独的文件夹。landscape和next是同层级的目录。）

7 自动化部署

这一步是我最陌生的，但配置过程很顺利，没有遇到啥问题。
https://hexo.io/zh-cn/docs/one-command-deployment

7.1 安装插件

$ npm install hexo-deployer-git --save

7.2 修改配置 _config.yml
7.3 执行部署

然后部署 执行命令：$ hexo deploy
期间命令行会要求你输入登陆github的username，password
一次输入就完成了

7.4 使用 .travis.yml

https://hexo.io/zh-cn/docs/github-pages
follow 其中的2到11步。在push到gitub仓库前，记得在新建好gh-pages分支。

检查

之后使用hexo new asd，生成一篇title，在source文件夹下找到对应刚生成的文章asd
编辑。完成后，执行 hexo deploy，过一会儿github上repo里应该就会有更新了。
ps: 如果没有，可能要

npm run build 重新构建

hexo clean 清除缓存

再deploy

over，yes！
